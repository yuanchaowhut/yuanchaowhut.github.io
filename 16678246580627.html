背景

在跟着做基于Vue3最新标准，实现后台前端综合解决方案项目时，第4章节里边有一个scss与js共享变量的知识点，按照教程中所说去做，中途遇到了一些坑点，在这里做一下记录。

实现方案

scss 文件与js文件共享变量实现方案步骤如下：

  1. 在 variables.scss 中配置要导出的变量

    // sidebar
$menuText: #bfcbd9;
$menuActiveText: #ffffff;
$subMenuActiveText: #f4f4f5;

$menuBg: #304156;
$menuHover: #263445;

$subMenuBg: #1f2d3d;
$subMenuHover: #001528;

$sideBarWidth: 210px;

// https://www.bluematador.com/blog/how-to-share-variables-between-js-and-sass
// JS 与 scss 共享变量，在 scss 中通过 :export 进行导出，在 js 中可通过 ESM 进行导入
:export {
  menuText: $menuText;
  menuActiveText: $menuActiveText;
  subMenuActiveText: $subMenuActiveText;
  menuBg: $menuBg;
  menuHover: $menuHover;
  subMenuBg: $subMenuBg;
  subMenuHover: $subMenuHover;
  sideBarWidth: $sideBarWidth;
}

  2. 在vue文件的script中导入变量并在template中使用

    <template>
  <div class="app-wrapper">
    <!-- 左侧 menu -->
    <sidebar id="guide-sidebar"
             class="sidebar-container"
             :style="{ backgroundColor: variables.menuBg }"
    />
    <div class="main-container">
      <div class="fixed-header">
        <!-- 顶部的 navbar -->
        <navbar />
      </div>
      <!-- 内容区 -->
      <app-main />
    </div>
  </div>
</template>

<script setup>
import {} from 'vue'
import Sidebar from './sidebar'
import Navbar from './navbar'
import AppMain from './appmain'
import variables from '@/styles/variables.scss'
console.log(variables)  // 输出空对象 {}

</script>

<style lang="scss" scoped>
@import '~@/styles/mixin.scss';
@import '~@/styles/variables.scss';

.app-wrapper {
  @include clearfix;
  position: relative;
  height: 100%;
  width: 100%;
}

.fixed-header {
  position: fixed;
  top: 0;
  right: 0;
  z-index: 9;
  width: calc(100% - #{$sideBarWidth});  /*这里可以正常引用*/
}
</style>

  3. 如果是vue2文件则如下方式使用

    <template>
      <el-menu
        :background-color="variables.menuBg"
        :text-color="variables.menuText"
      >
      </el-menu>
</template>

<script>
import variables from '@/styles/variables.scss'
//使用scss的变量，在data中定义也可以
export default {
  computed: {
    variables() {
      return variables
    }
  }
}
</script>

实际问题

按照上述流程编写代码后，发现script中引入的对象实际为空，console.log(variables) 打印结果为{}，通过网上查阅各种资料，最后将 variables.scss 重命名为 variables.module.scss 才解决问题。具体可以参考下面的资料。
