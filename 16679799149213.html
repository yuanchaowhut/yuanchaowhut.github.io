官方示例

如上图所示，官方示例中使用了 switcherIcon 这个 slot。然后使用了 antd 内置的向下箭头图标，展开和收起时通过旋转 90deg 来控制图标显示。

<template>
  <a-tree show-line :default-expanded-keys="['0-0-0']" @select="onSelect">
    <a-icon slot="switcherIcon" type="down" />
    <a-tree-node key="0-0" title="parent 1">
      <a-tree-node key="0-0-0" title="parent 1-0">
        <a-tree-node key="0-0-0-0" title="leaf" />
        <a-tree-node key="0-0-0-1" title="leaf" />
        <a-tree-node key="0-0-0-2" title="leaf" />
      </a-tree-node>
      <a-tree-node key="0-0-1" title="parent 1-1">
        <a-tree-node key="0-0-1-0" title="leaf" />
      </a-tree-node>
      <a-tree-node key="0-0-2" title="parent 1-2">
        <a-tree-node key="0-0-2-0" title="leaf" />
        <a-tree-node key="0-0-2-1" title="leaf" />
      </a-tree-node>
    </a-tree-node>
  </a-tree>
</template>

开发实践

实际开发中，由一个UI统一规范，要求必须使用它们的小图标。我们项目上使用的是 iconfont ，按照上面的方案换上去后尴尬的情况出现了，展开/收起 小图标全部显示相同，即收起时没有旋转 90deg 效果。

  <a-tree class="diy-tree" :show-icon="showIcon">
    .....
    <!--修改折叠图标，这里只定义1个展开即可，收起时通过旋转-90度实现的-->
    <em slot="switcherIcon" class="iconfont icon-csgl-nav-expand custom-tree-switcher"></em>
    ......      
    </a-tree>

F12 查看DOM结构如下:

经过对比，原来使用 iconfont 后，DOM结构中没有官方示例中 svg 标签了，故官方定义的针对 svg 的旋转样式就不起作用了。不过知道原因后，解决起来就简单了，没有旋转效果，我们自己定义在样式文件中声明一个就可以了。

// 展开样式
.ant-tree-treenode-switcher-open .custom-tree-switcher {
    transform: rotate(0deg) !important;
}

// 收起样式
.ant-tree-switcher_close .custom-tree-switcher {
    transform: rotate(-90deg) !important;
}

// 这个是优化样式，不属于更换折叠图标范畴
.ant-tree-node-content-wrapper {
    margin-left: -0.1rem !important;
}

扩展思考

实际开发中我们经常看到这样的设计图：

那么这种效果能不能实现？渲染这两个图标不能通过旋转 90deg 来实现，但是网上查阅资料，找到一个解决方案，其实和上面的实践过程大同小异，也是找到 close 和 open 这两个特定的样式，然后再给它进行相关设置。网上作者是用的本地图片，实际上我们也可以使用 iconfont 图标。

:global{
    .ant-tree-switcher_close{
        background: url('../../assets/browse/down.png') no-repeat;
    }
    .ant-tree-switcher_open{
        background: url('../../assets/browse/up.png') no-repeat;
    }
}
